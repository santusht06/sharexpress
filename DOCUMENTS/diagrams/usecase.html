📋 Use Case Descriptions

AUTH Authentication & Account

  • Send OTP: Request 6-digit OTP via email
  • Verify OTP: Validate OTP and create session
  • OAuth Login: Sign in with Google/GitHub
  • Guest Session: 24-hour temporary access
  • Update Profile: Change name and picture
  • Delete Account: Permanent data removal

SESSION Session Management

  • Generate QR: Create shareable QR code
  • Scan QR: Join session via camera
  • Create Session: Initialize file sharing
  • Search User: Find by email address
  • View Sessions: List sent/received/active
  • Revoke Access: Terminate session early
  • Set Permissions: Upload/Download/Edit

FILES File Operations

  • Initialize Upload: Get presigned URL
  • Upload to S3: Direct client upload
  • Complete Upload: Confirm and finalize
  • Download File: Secure presigned URL
  • List Files: View session contents
  • Delete File: Remove from session
  • View Metadata: Size, type, timestamp

SYSTEM System Operations

  • Auto-Delete: 30-day file expiration
  • Monitor Health: API uptime checks
  • View Analytics: Usage statistics
  • Rate Limiting: Prevent abuse
  • Circuit Breaker: Fault tolerance
  • Observability: Metrics & tracing

👥 Actor Descriptions

👤 REGISTERED USER

  • Authenticated via OTP or OAuth
  • Full access to all features
  • Persistent profile and history
  • Can create and manage QR codes
  • Email-based session creation
  • Profile picture management

👥 GUEST USER

  • Anonymous 24-hour session
  • Can join via QR scan
  • Upload and download files
  • No persistent history
  • Limited to active sessions
  • No QR code generation

🔐 ADMIN

  • System monitoring access
  • Analytics and metrics
  • Rate limit configuration
  • User management
  • Health check monitoring
  • Incident response

⚙️ EXTERNAL SYSTEMS

  • OAuth providers (Google, GitHub)
  • AWS S3 / MinIO storage
  • MongoDB Atlas database
  • Redis cache
  • Email service (OTP delivery)
  • Monitoring (Prometheus, Jaeger)

🔗 Relationship Types

Association (—)

  • Actor directly performs use case
  • Example: User → Login with OTP
  • Solid line connection

Include (---→)

  • Use case always includes another
  • Example: Verify OTP includes Logout
  • Dashed line with arrow

Extend (---→)

  • Optional additional behavior
  • Example: Update Profile extends to Upload Picture
  • Dashed line with arrow

Generalization (→)

  • Specialization of use case
  • Example: OAuth methods extend Login
  • Used for inheritance
